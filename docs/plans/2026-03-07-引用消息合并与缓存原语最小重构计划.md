## 缓存/持久化原语最小重构（不改变行为）

为减少重复实现与维护成本，仅在“底层原语”层收敛，保留领域语义边界：

### 任务 1：内存 TTL/LRU 缓存原语

- 文件：`src/cache/memory-ttl-cache.ts`
- API：
  - `createBoundedCache<V>({ maxKeys, defaultTtlMs, eviction: 'lrutime' | 'expiry' })`
  - 接口 `KeyValueCache<K extends string, V>`：`get/set/delete/has/size/sweep(now)`
- 行为：
  - Map + 元数据（`expiresAt`、`lastAccessAt`）；先清理过期，再按策略淘汰
  - 无定时器；调用方按需 `sweep()`
- 单测：`tests/unit/cache-memory-ttl.test.ts`（TTL、淘汰、sweep）

### 任务 2：文件持久化适配器（原子写）

- 文件：`src/cache/file-kv-store.ts`
- API：

  - `load(log?) => Record<string,V>`（坏文件→空映射+warn）
  - `flush(data, log?)`（临时文件 + rename）
  - `atomicUpsert(mutator, log?)`
- 单测：`tests/unit/cache-file-kv.test.ts`（坏文件容错、原子写、upsert 顺序）

## 风险与缓解（更新）

- 持久化适配器缺陷可能影响恢复 → 原子写 + 容错加载 + 单测
- 淘汰策略变更可能影响命中 → 保留“按会话分桶 + 当前参数”的实现
- 避免引文内容误持久化 → 卡片内容索引默认仅内存；未来如需持久化须显式 opt-in 且最小化内容
- journal 含正文 → 默认 7 天 TTL；README 明示；云/多租户环境注意磁盘与敏感信息治理
