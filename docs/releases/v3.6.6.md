# v3.6.6 发布说明

本次 patch 版本聚焦 **DingTalk AI Card 的回复稳定性与状态栏准确性**，并补齐表单互动卡片的用户文档和可发布模板资产。

**最新版本入口**：[`latest.md`](./latest.md)

## 🛠 修复与稳定性

* **避免晚到工具错误覆盖正常答案**
  * [`PR #587`](https://github.com/soimy/openclaw-channel-dingtalk/pull/587)（by @CRISPpp）在已收到正常最终答案后忽略随后到达的非终止性纯文本工具错误，确保 AI Card 保留有效回复。
  * 普通后续 final 仍可更新答案，首条回复本身为错误时也会继续正常展示。

* **首张 AI Card 显示配置模型**
  * [`PR #584`](https://github.com/soimy/openclaw-channel-dingtalk/pull/584) 修复首次回复状态栏缺少模型名的问题，并保留 runtime 选定模型的权威覆盖语义。
  * task metadata 现在按账号、agent 与会话隔离，避免同一会话切换 agent 时模型和 effort 状态串线。

## 📚 文档与发布资产

* **补齐表单互动卡片使用指南与模板**
  * [`PR #585`](https://github.com/soimy/openclaw-channel-dingtalk/pull/585)（by @BenGuanRan）新增 `dingtalk_ask_user_question` 的启用、鉴权、回调和互斥行为说明。
  * DingTalk ask-user 卡片模板现随 npm 包发布，便于安装后直接导入和配置。
