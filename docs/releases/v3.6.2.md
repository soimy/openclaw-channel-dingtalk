# v3.6.2 发布说明

本次 patch 版本聚焦 **npm / ClawHub 包可安装性修复，以及运行时安全扫描兼容性**。

**最新版本入口**：[`latest.md`](./latest.md)

## 🛠 修复与稳定性

* **修复 npm 包缺少编译后 OpenClaw runtime 产物的问题**
  * [`commit 964d48c`](https://github.com/soimy/openclaw-channel-dingtalk/commit/964d48c06d8ae4922245955a812f7f4873a585a9)（by @soimy）将发布入口切换到 `dist/index.js`，新增 runtime build、类型产物和包内容校验，修复 OpenClaw 2026.5.x 从 npm 安装插件时缺少编译后 runtime 文件的问题。

* **移除 runtime 包中的进程执行路径**
  * 同一修复移除了 SecretInput 的 `exec` 来源与对应文档入口，只保留 `env` / `file` 引用，避免安装安全扫描因 runtime 内含进程执行调用而阻断插件安装。

## 🧩 发布链路

* **补齐发布前包内容校验**
  * 新增 `pnpm run build` 与 `pnpm run pack:check`，CI 在 npm / ClawHub 发布前会确认 `dist/index.js`、`dist/index.d.ts` 与 `openclaw.plugin.json` 均进入发布包，并检查 runtime 产物不包含进程执行调用。
